# Navi 私人书签导航页

Navi 是一个放在你自己设备上的书签导航面板。它不需要账号、不需要后端服务，书签、分类、设置和操作日志都保存在当前浏览器本地。

你可以把它当作：

- 日常打开网站的首页
- 浏览器书签的可视化整理工具
- 手机或平板上的 PWA 书签 App
- 一个可以离线使用、可导入导出、可撤销操作的私人导航页

如果你是第一次使用，建议先从下面的“最快开始”看起。

---

## 最快开始

### 方式一：直接打开

双击项目里的 `index.html`，浏览器会直接打开 Navi。

这种方式最简单，适合先体验界面、手动添加书签、导入浏览器导出的书签文件。因为它运行在 `file://` 本地文件环境，PWA 离线安装和浏览器实时书签同步不会启用。

### 方式二：作为浏览器扩展使用

如果你希望同步 Chrome 或 Edge 的书签，推荐用扩展方式打开。

1. 打开扩展管理页：
   - Chrome：`chrome://extensions`
   - Edge：`edge://extensions`
2. 开启“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择这个项目文件夹。
5. 点击工具栏里的 Navi 图标打开页面。
6. 进入“设置 → 通用 → 浏览器书签同步”，选择来源和同步规则。

扩展同步是只读的：Navi 会读取浏览器书签并生成自己的面板数据，不会反向修改浏览器原生书签。

### 方式三：作为手机 PWA 使用

把这个文件夹部署到支持 HTTPS 的静态网站，例如 GitHub Pages、Vercel、Netlify，然后在手机浏览器中打开并选择“添加到主屏幕”。

PWA 适合移动端长期使用，也支持离线访问。请注意：卸载 PWA 或清除站点数据前，建议先导出备份。

---

## 第一次打开后建议做什么

1. 先确认打开方式

如果你想同步 Chrome 或 Edge 书签，请一定从扩展图标打开 Navi。直接双击 `index.html` 和扩展页面是两份不同的数据。

2. 导入或同步书签

你可以点击顶部“导入”，选择浏览器导出的书签 HTML 文件。也可以在设置中开启浏览器书签同步。

3. 选择分类显示方式

进入“设置 → 仪表盘”，可以在标签栏和侧边栏分类之间切换。

4. 调整外观和小组件

进入“设置 → 外观”设置主题、背景、玻璃效果；进入“设置 → 仪表盘”选择时钟、搜索、天气、日历、常用、最近、内网监控等小组件。

5. 导出备份

整理完成后，建议通过“更多 → 导出书签”保存一份 HTML 备份。

## 书签导入与同步规则

Navi 支持多种来源：

- Chrome：扩展环境下可读取书签
- Edge：扩展环境下可读取书签
- Safari：推荐通过 Safari 导出的 HTML 文件导入
- 其他浏览器：使用书签 HTML 文件导入

在“浏览器书签同步”或导入预览中，可以选择同步规则：

| 规则 | 适合场景 |
|---|---|
| 合并 | 把新来源并入现有面板，重复 URL 会跳过或更新已有书签 |
| 按来源独立 | 不同浏览器来源分开管理，分类名前会加来源前缀 |
| 替换当前来源 | 只清除当前来源的旧书签，再重新生成该来源 |
| 替换全部 | 清空 Navi 本地全部书签和分类，再按当前来源重建 |

如果你同时使用 Chrome、Edge、Safari，建议先用“按来源独立”观察效果，再决定是否改为合并。

## 常用功能

### 书签管理

- 添加、编辑、删除书签
- 自动抓取标题和简短描述
- 支持分类、标签、描述
- 支持卡片拖拽排序
- 支持批量选择和批量删除

### 分类管理

- 新建分类
- 重命名分类
- 删除分类
- 拖拽调整分类顺序
- 标签栏和侧边栏两种分类布局

### 描述摘要

书签描述可以手动填写，也可以点击“自动”生成。更多菜单里的“生成描述摘要”可以批量处理：

- 全部书签
- 当前分类
- 指定分类
- 只补空描述
- 重新生成全部描述

如果设置了 Claude 或 OpenAI API Key，会优先读取网页内容并调用 AI 总结；没有设置时会使用本地规则生成简短描述。

### 操作日志与撤销

每次重要修改都会记录到“设置 → 操作日志”。日志可以撤销近期操作，也可以自动过期清理。

默认日志保留 2 天，你可以改为：

`立即、1 天、2 天、3 天、5 天、7 天、14 天、30 天、不删除`

这样可以减少本地缓存占用。

### 回收站

删除书签会先进入回收站。你可以恢复、永久删除，或设置回收站保留时长。

### 链接健康检查

“更多 → 检查链接有效性”可以检查书签是否可访问。

扩展环境下结果更准确；普通网页或 PWA 会受浏览器跨域限制，可能只能判断大致连通性。

### AI 分类建议

“更多 → AI 分类建议”会根据标题、URL、描述推荐分类。默认使用本地规则，不需要联网、不需要 API Key。

如果你在“设置 → 监控 & AI”中选择 Claude 或 OpenAI 并填入 API Key，会得到更智能的分类建议。Key 只保存在本机浏览器中。

### 内网监控

小组件里可以开启“状态监控”，用于展示 NAS 或内网服务状态。未配置时会显示示例数据，方便你理解它的效果。

如果要连接真实 NAS，推荐使用 Glances 或通用 JSON 接口。公网部署时不要暴露未鉴权的内网监控接口。

## 数据保存在哪里

Navi 的数据保存在浏览器本地 `localStorage`。

这意味着：

- 不会自动上传到服务器
- 不同浏览器之间默认互不相通
- 扩展页面、本地文件页面、PWA 页面是三份独立数据
- 清除浏览器站点数据会清除 Navi 数据

常见例子：

- 从 Chrome 扩展图标打开：数据属于 `chrome-extension://...`
- 双击 `index.html` 打开：数据属于 `file://...`
- 从网站/PWA 打开：数据属于该 HTTPS 域名

如果你发现“刚才整理好的书签不见了”，通常是因为换了打开方式。请回到原来的入口，或者用导出/导入迁移数据。

## 备份与迁移

推荐养成导出备份的习惯：

1. 点击右上角“更多”。
2. 选择“导出书签”。
3. 保存生成的 HTML 文件。
4. 在另一台设备或另一个浏览器中点击“导入”。

这个 HTML 文件兼容常见浏览器书签格式，也方便长期保存。

---

## PWA 与离线使用

PWA 模式需要 HTTPS。通过 `http://` 或本地 `file://` 打开时，Service Worker 不会启用。

如果你修改了代码但 PWA 里没有更新，通常是离线缓存还在使用旧版本。需要更新 `sw.js` 顶部的缓存版本号，并确保新增文件被加入缓存列表。

## 文件结构

你不需要理解这些文件也可以正常使用 Navi。下面内容主要给想改代码的人看。

```text
index.html              页面结构和弹窗
css/app.css             全部样式
js/state.js             默认状态和设置
js/utils.js             保存、读取、通用工具
js/render.js            书签和分类渲染
js/bookmarks.js         书签增删改、标题/描述生成
js/categories.js        分类管理
js/import-export.js     浏览器书签 HTML 导入导出
js/chrome-sync.js       Chrome / Edge / Safari 来源同步逻辑
js/settings.js          设置面板
js/widgets.js           顶部小组件
js/oplog.js             操作日志和撤销
js/trash.js             回收站
js/health.js            链接检查
js/suggest.js           AI 分类建议
js/monitor.js           内网监控
js/pwa.js               PWA 注册
js/app.js               启动入口
sw.js                   PWA 离线缓存
manifest.json           浏览器扩展清单
manifest.webmanifest    PWA 清单
background.js           扩展后台脚本
```

项目不需要构建工具。修改后直接刷新页面即可；如果是扩展模式，需要在扩展管理页点击刷新扩展。

## 安全提醒

- API Key 只建议填在你信任的本机浏览器中。
- 不要把包含个人书签数据的导出文件发给不可信的人。
- 不要把未鉴权的 NAS/Glances/内网服务接口暴露到公网。
- 清除浏览器站点数据、卸载 PWA、删除扩展，都可能导致本地数据丢失；操作前请先导出备份。
